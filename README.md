# 🏰 رحلة العائلة — Family Quest

منصّة مكافآت عائلية خاصة تُحفّز الأطفال على بناء عادات إيجابية من خلال الأنشطة، الإنجازات، النقاط، المكافآت، السلاسل اليومية، والمنافسة الودّية.

A private family rewards platform that motivates children to build positive habits through activities, achievements, points, rewards, streaks, and friendly competition. Arabic-first (RTL), mobile-first, zero backend.

---

## ✨ المميزات | Features

- **لوحة العائلة** — نقاط كل طفل، الترتيب، التقدّم الأسبوعي والشهري، السلاسل، آخر الأحداث.
- **ملفات الأطفال** — صورة رمزية، نقاط مكتسبة/مصروفة، رصيد، أنشطة، أوسمة، سجل المكافآت.
- **كتالوج الأنشطة** — أربع فئات (المنزل، التعلّم، الرياضة، الدين والأخلاق)، قابلة للتعديل بالكامل.
- **نظام الموافقات** — الطفل يُكمل النشاط ← ينتظر موافقة ولي الأمر ← تُضاف النقاط بعد الموافقة فقط.
- **متجر المكافآت** — رقمية، تجارب، طعام، مكافآت خاصة. طلب ← موافقة ← خصم تلقائي.
- **الأوسمة والإنجازات** — ٨ أوسمة مع رسوم متحركة.
- **نظام السلاسل** — يومية، قراءة، رياضة، مع إعادة التصفير عند الانقطاع.
- **لوحة الترتيب** — أسبوعي/شهري/كلّي، برسائل تشجيعية ودّية.
- **التحديات العائلية** — أسبوع القراءة، تحدي اللياقة، وغيرها، بنقاط إضافية.
- **التقارير والإحصائيات** — ملخّص أسبوعي وشهري لكل طفل.
- **لوحة الإدارة الكاملة** — إضافة/تعديل/حذف لكل شيء دون لمس الكود.
- **حماية البيانات** — حفظ تلقائي، تصدير/استيراد JSON، تأكيد قبل الحذف.
- **التصميم** — وضع نهاري وليلي، رسوم متحركة، أزرار كبيرة، احتفال بالـ confetti، مؤثرات صوتية.

## 🚀 النشر على GitHub Pages | Deploy to GitHub Pages

1. أنشئ مستودعاً جديداً على GitHub (مثلاً `family-quest`).
2. ارفع كل ملفات هذا المجلّد إلى المستودع (مع الحفاظ على البنية):
   ```
   index.html
   .nojekyll
   css/styles.css
   js/*.js
   ```
   عبر الواجهة: **Add file → Upload files** ثم اسحب الملفات.
   أو عبر Git:
   ```bash
   git init
   git add .
   git commit -m "Family Quest"
   git branch -M main
   git remote add origin https://github.com/<username>/family-quest.git
   git push -u origin main
   ```
3. من المستودع: **Settings → Pages**.
4. تحت **Branch** اختر `main` والمجلّد `/ (root)` ثم **Save**.
5. انتظر دقيقة، وسيظهر الرابط:
   `https://<username>.github.io/family-quest/`

> ملف `.nojekyll` مهم: يمنع GitHub Pages من تجاهل مجلّد `js/`. لا تحذفه.

التطبيق يعمل فوراً بعد الرفع، ببيانات تجريبية جاهزة لطفلين.

## 🔑 الدخول | Access

- **ولي الأمر:** اضغط على بطاقة «ولي الأمر» وأدخل الرمز. الافتراضي: **`1234`** (يمكن تغييره من الإدارة ← البيانات).
- **الأطفال:** يضغطون على صورتهم الرمزية مباشرةً — بلا رمز.

## 🗂️ بنية الملفات | File Structure

```
family-quest/
├── index.html          # نقطة الدخول، RTL، الخطوط
├── .nojekyll           # ضروري لـ GitHub Pages
├── css/
│   └── styles.css      # نظام التصميم، السمات، التحريك
└── js/
    ├── store.js        # البيانات الافتراضية + التخزين (LocalStorage)
    ├── logic.js        # القواعد: نقاط، موافقات، سلاسل، أوسمة، إحصائيات
    ├── ui.js           # المؤثرات: confetti، صوت، توست، نوافذ
    ├── views.js        # البوابة، الهيكل، لوحة الطفل
    ├── views2.js       # الأنشطة، المكافآت، الأوسمة، الترتيب، لوحة ولي الأمر، التقارير
    ├── admin.js        # لوحة الإدارة الكاملة (CRUD + البيانات)
    └── app.js          # نقطة التشغيل
```

## 🛠️ التقنيات | Tech

HTML + CSS + JavaScript خالص. لا إطار عمل، لا أدوات بناء، لا خادم. كل البيانات في `localStorage` تحت المفتاح `familyQuest:v1`، مع نسخة احتياطية تلقائية تحت `familyQuest:backup`.

---

## 💾 النسخ الاحتياطي | Backups

- **تلقائي:** يُحفظ كل تغيير فوراً في المتصفح.
- **يدوي:** الإدارة ← البيانات ← «تصدير JSON» / «استيراد JSON».
- ننصح بتصدير نسخة دورياً، خصوصاً قبل مسح بيانات المتصفح.
